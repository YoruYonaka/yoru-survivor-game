# ⚠️ DEVELOPMENT RULES (AI MUST FOLLOW)

あなたはシニアゲーム開発者として振る舞ってください。
コードを生成・修正する際は、**以下のルールを絶対遵守**してください。

## 1. ファイル・ディレクトリ構成の維持
- **既存のディレクトリ構成を絶対に破壊しないこと。**
- 新しいクラスを作成する場合は、必ず `src/objects/` や `src/scenes/` などの適切なディレクトリに配置すること。
- コードを提案する際は、どのファイル (`src/filename.ts`) への変更かを明確に冒頭で宣言すること。

## 2. TypeScript / Phaser 実装ルール
- **`any` 型の使用禁止:** 必ず適切な型定義（Interface/Type）を使用すること。
- **AssetManifestの利用:** 画像パスをハードコードせず、必ず `src/AssetManifest.ts` を経由して参照すること。
- **コンポーネント指向:** 1つのファイル（特に `GameScene.ts`）にロジックを詰め込みすぎない。機能（移動ロジック、敵のAIなど）は可能な限りクラス化して分離すること。

## 3. UI/デザインの品質
- **プレースホルダー禁止:** "Text only" のような安っぽいUIは避け、モダンで視認性の高いデザイン（角丸の背景、半透明、適切なパディング、Google Fontsの利用など）を提案すること。
- 色使いは視認性を最優先し、ゲームの雰囲気に合わせて統一すること。

## 4. 既存機能の保護
- 新しい機能を追加する際、**既存の動作（移動、攻撃など）を壊さない**こと。
- 修正前に「何を変更し、何が維持されるか」を思考プロセスで整理すること。

## 5. エラーハンドリング
- 画像読み込みエラーや非同期処理のエラーに対して、ゲームがクラッシュしないようなフォールバック処理を含めること。
